# 顶栏设计与封装规范

## 1. 概述
在客户端软件架构中，顶栏（Top Bar / Header）是提升软件辨识度并收敛通用操作项的核心组件。为了确保团队旗下或后续所有子软件的顶栏风格和交互逻辑达到统一的标准，特将原项目主界面的顶栏以及通用的业务弹窗（如“赞赏”、“关于”）抽离为独立的高度自治可复用组件 `top_bar.py`，并设立本规范文档供其他项目作为参考模板。

## 2. 视觉设计原则

### 2.1 基础布局
- **高度**：固定高度为 `50px`，能够保证充足的点击区域而不显得拥挤。
- **背景色**：采用深色渐变背景横滑过渡设计，例如 `qlineargradient(x1:0, y1:0, x2:1, y2:0, stop:0 #1a2a3a, stop:0.5 #2c3e50, stop:1 #1a2a3a)`，以强调其在视觉层级的优先级。
- **底部边框**：拥有亮色底边（如 `2px solid #409eff`），将顶栏与主工作区进行平滑的物理隔离。

### 2.2 标题排版
- **自适应与间距**：顶部元素左右两侧应保留 `20px` 边距。
- **标题文字**：位于最左侧，颜色为 `White`，加粗，`16px` 字号。字间距应使用 `letter-spacing: 2px` 以凸显高级感。

### 2.3 操作按钮
- **排列位置**：右侧对齐，依次排布关键性通用按钮（如“赞赏支持”、“关于”按钮）。中间通过弹簧 (`Stretch`) 将两侧排开。
- **色彩搭配与层级**：
  - **重要按钮**（例如：赞赏/打赏）：建议采用红/橙系热色调渐变（如 `#e74c3c` 到 `#c0392b`）。
  - **辅助按钮**（例如：关于/设置）：建议采用蓝/青系冷色调渐变（如 `#5dade2` 到 `#3498db`）。
- **交互状态**：所有的按钮在 `Hover` 时需有平滑的颜色反转提亮过渡呈现；统一设置手指光标 (`PointingHandCursor`)。

## 3. 组件级高度自治（封装策略）

### 3.1 弹窗内聚设计
由于“赞赏作者”与“软件关于”界面是全系列软件所通用的标准界面，为了最大化减少代码冗余，**这两个弹窗的定义 (`DonateDialog` 及 `AboutDialog`) 全部内聚在 `top_bar.py` 中**。
顶栏无需主程序介入，即可在按钮点击时自动弹出这些信息。这让组件达到了“插上即用”的开箱即用效果。

**内部处理逻辑示例**：
```python
    def show_donate(self):
        """显示赞赏对话框，并向外部广播事件"""
        DonateDialog(self).exec()
        self.donate_clicked.emit()
```

### 3.2 自闭环样式
所有的 CSS/QSS 应在该子组件内部隔离。使用 `self.setStyleSheet()` 并附带 `Id` 选择器防止污染外部环境（例如 `QFrame#headerFrame`）。

### 3.3 信号隔离广播
尽管顶栏包揽了点击弹窗的任务，但组件依旧保留并广播 `donate_clicked` 与 `about_clicked` 信号。这是为了防止未来主程序在用户点击赞赏时，需要做类似“埋点统计”、“暂停其他线程活动”等额外订阅处理。

## 4. 如何在其他参考项目中使用

### 4.1 资源依赖约束
独立顶栏自带的弹窗中默认包含了二维码图片，请在将 `top_bar.py` 移植到各类新项目时，务必**一同拷贝以下通用图片资源**，并与 `top_bar.py` 放在同一代码级目录下：
- `donate_qr.png` (原赞赏收款码)
- `official_qr.png` (原公众号名片码)

*(注：如果目标工程忘记存放图片资源，由于代码中存在 `os.path.exists()` 资源保护机制，软件不会崩溃，但这块区域将无法显示二维码，或者显示默认的文字提示。)*

### 4.2 接入代码
将 `top_bar.py` 以及相关图片拷入你的新项目中。然后在新的主窗口类初始化时，只需要写入下面几行代码即可完成该顶栏组件的高级复用：

```python
# 导入顶栏及可能需要外部手动调用的弹窗
from top_bar import TopBar, DonateDialog

# 在主界面的布局组合中...
# 实例化顶栏，组件内部已自动接管了按钮点击和弹窗处理
self.header = TopBar("您的新软件名称")

# 将顶栏插入到垂直布局最上方
main_layout.addWidget(self.header)
```
*(注意：主界面不再需要 `.connect` 来接管赞赏按钮。除非你想在点击后执行除了弹窗以外的其他逻辑才需要去 `connect` 这些信号。)*

## 5. 扩展指导
如果你后续需要在目前的代码架构上添加类似“主题切换”、“快速设置”功能：
1. **构建按钮**：在 `TopBar` 类的 `initUI` 中添加控件，并设置独立的 QSS Id 以及渐变风格，与右侧现有按钮平行放置。
2. **定义或绑定弹窗**：如果是其他项目中通用的全生命周期窗口，直接将弹窗类写进 `top_bar.py` 并在点击槽中 `exec()` 调用。
3. **新增事件抛出**：依然记得定义新的 `pyqtSignal` 并在方法结尾抛出，保证组件与主系统间的低耦合通信机制。
